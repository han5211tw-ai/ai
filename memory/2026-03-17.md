## 經驗記錄

- **代碼清理要徹底**：移除功能時要同時清理 HTML、CSS、JavaScript，避免殘留
- **版本維護**：檔案頭的版本日期要隨修改更新
- **部署相容性**：避免硬編碼絕對路徑，使用相對路徑或環境變數
- **啟動階段保護**：非必要的初始化動作要加入錯誤處理，避免影響啟動
- **JavaScript 嚴格模式**：重複宣告 `const` 會導致語法錯誤，整個腳本無法執行
- **除錯技巧**：當頁面顯示「載入中」但沒有錯誤訊息時，可能是 JavaScript 語法錯誤導致腳本完全沒有執行
- **資料同步**：新舊系統並行時，要確保資料同步機制正常運作

## 經驗記錄（追加）

- **Telegram 群組 ID 取得：** 透過 `https://api.telegram.org/bot<token>/getUpdates` 可以取得群組 ID，格式為負數（如 `-5232179482`）
- **背景執行緒機制：** `threading.Thread` + `daemon=True` + `.start()` 可以讓 HTTP 回應立即返回，不用等待外部 API
- **資料一致性：** 前端選項的 value 必須與資料庫實際儲存的值一致，否則會導致查詢失敗
- **通知機制分層：**
  - 新需求提交 → 通知老闆/會計（個人）
  - 狀態變更（已採購/已調撥）→ 通知工作群組（全體同仁）
