## 今日完成項目

### 1. LINE 官方帳號回覆表系統 ✅

**任務**: 建立 LINE 官方帳號回覆表功能，取代雲端文件填寫

**完成內容**:
- 資料表 `line_replies` 建立（SQLite）
- 6組 API 端點：
  - `GET /api/line-replies` - 查詢列表（分頁、篩選、搜尋）
  - `POST /api/line-replies` - 新增記錄
  - `GET /api/line-replies/<id>` - 取得單筆
  - `PUT /api/line-replies/<id>` - 更新記錄
  - `DELETE /api/line-replies/<id>` - 刪除記錄
  - `GET /api/line-replies/stats/*` - 門市/人員統計
- 前端頁面（繼承 base.html）：
  - `/line_replies` - 列表頁
  - `/line_replies_edit` - 新增/編輯頁
- 選單整合：工作紀錄區塊下，班表上方
- 權限：全部人員可見

**樣式調整**（與銷貨輸入頁面一致）：
- 按鈕：`padding: 10px 20px`, `font-size: 0.95em`, `font-weight: 500`
- 表格：有框線樣式，th 背景 `rgba(250, 191, 19, 0.1)`，文字顏色 `#FABF13`
- 表單：使用完整工作區寬度

**重要教訓**:
- 初期未使用 base.html 模板，導致樣式不一致、登入機制不相容
- 重新改寫為 Jinja2 模板格式後才符合系統規範
- **規則確立**：未來所有新頁面必須繼承 base.html，使用共用樣式與元件
