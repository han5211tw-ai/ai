# 2026-03-16 工作記錄

## Google 商家五星評論自動化系統

### 背景
老闆希望自動追蹤三個門市（豐原、潭子、大雅）的 Google 商家五星好評數量變化，從 2026/4/1 開始記錄。

### 解決方案
利用 Google 商家通知信寄到專用信箱，AI 讀取解析後寫入資料庫。

---

## 已完成項目

### 1. 郵件解析腳本
- **檔案**: `/Users/aiserver/srv/parser/google_reviews_parser.py`
- **功能**:
  - 使用 Himalaya CLI 讀取 `ai@computershop.cc` 未讀郵件
  - Regex 解析門市名稱、評論者、星等（1-5星）
  - 寫入 `google_reviews` 資料表
  - 更新 `google_reviews_stats` 統計
  - 處理後移動郵件到垃圾桶

### 2. 資料表設計
```sql
-- 單筆評論明細（記錄所有星等）
google_reviews (
    id, store_name, reviewer_name, review_date,
    star_rating, review_snippet, email_subject,
    email_received_at, processed_at, parse_method
)

-- 門市統計快取
google_reviews_stats (
    store_name, five_star, four_star, three_star,
    two_star, one_star, total_reviews, avg_rating, updated_at
)
```

### 3. API Endpoint
- `/api/store/reviews` - 相容舊版，回傳五星數量
- `/api/google-reviews` - 詳細評論列表（支援篩選）
- `/api/google-reviews/stats` - 各門市統計

### 4. 頁面更新
- **檔案**: `/Users/aiserver/.openclaw/workspace/dashboard-site/store.html`
- 五星好評統計區塊顯示三店數量
- 新增「最近五星評論」區塊，顯示最新 5 筆

### 5. 排程設定
- **時間**: 每天 00:00
- **指令**: `python3 google_reviews_parser.py`
- **狀態**: 已設定但註解，待 3/31 啟用
