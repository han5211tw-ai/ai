### ERP v2 登入錯誤修復（2026-03-28 14:52）

**問題：**
- 登入時出現「伺服器錯誤」
- 錯誤日誌顯示：`sqlite3.OperationalError: no such column: s.employee_id`

**原因：**
- `auth_verify` 函數的 SQL 查詢中，`staff` 和 `staff_passwords` 的 JOIN 條件是用 `name` 欄位比對
- 但 `staff` 資料表的主鍵是 `staff_id`，不是 `name`
- 應該用 `staff_id` 進行 JOIN

**修復：**
- 修改 `/Users/aiserver/srv/web-site/computershop-erp/app.py` 第 138 行
- 將 `ON s.name = sp.name` 改為 `ON s.staff_id = sp.staff_id`

### Gunicorn 服務統一管理（2026-03-28 14:55-15:00）

**原本狀態：**
| 服務 | Port | 管理方式 |
|-----|------|---------|
| ERP v1 | 3000 | launchctl |
| ERP v2 | 8800 | 手動 daemon |
| 官網 | 8000 | 獨立運行 |

**問題：**
- ERP v2 使用 `gunicorn --daemon` 手動啟動，關機後需手動重啟
- 官網也是獨立運行，無法統一管理

**解決方案：**
建立三個 launchctl plist 設定檔：

1. **com.dashboard.gunicorn** (port 3000) - 原本就有
   - 路徑：`~/Library/LaunchAgents/com.dashboard.gunicorn.plist`
   - 工作目錄：`~/.openclaw/workspace/dashboard-site/`

2. **com.computershop.erpv2** (port 8800) - 新建
   - 路徑：`~/Library/LaunchAgents/com.computershop.erpv2.plist`
   - 工作目錄：`~/srv/web-site/computershop-erp/`
   - Python：`/opt/homebrew/bin/python3`

3. **com.computershop.website** (port 8000) - 新建
   - 路徑：`~/Library/LaunchAgents/com.computershop.website.plist`
   - 工作目錄：`~/srv/web-site/computershop-web/`
   - Python：`~/srv/web-site/computershop-web/venv/bin/python`

**管理指令：**
```bash
# 查看狀態
launchctl list | grep -E "(dashboard|computershop)"

# 啟動/停止
launchctl start com.dashboard.gunicorn      # ERP v1 (3000)
launchctl start com.computershop.erpv2      # ERP v2 (8800)
launchctl start com.computershop.website    # 官網 (8000)

launchctl stop com.dashboard.gunicorn
launchctl stop com.computershop.erpv2
launchctl stop com.computershop.website
```

**優點：**
- ✅ 開機自動啟動
- ✅ 當機自動重啟（KeepAlive）
- ✅ 統一管理方式
- ✅ 日誌集中輸出到 `/Users/aiserver/srv/logs/`
