## 產品建檔頁面修復

### 問題回報
Alan 回報產品建檔頁面有兩個問題：
1. 用 5211 登入後，建檔人員欄位沒有帶入姓名
2. 「建立產品資料」按鈕沒有反應

### 問題分析

#### 問題 1：按鈕沒反應
- `submitForm()` 函數中使用了 `document.getElementById('unit')`，但頁面中根本沒有這個欄位
- 導致 JavaScript 拋出錯誤，整個函數中斷執行

#### 問題 2：建檔人員欄位空白
- 頁面載入時 `checkAuth` 可能還沒準備好
- 原本的程式碼沒有等待 `auth_ui.js` 載入完成

### 修復內容

#### 修復 1：移除不存在的欄位引用
```javascript
// 修改前
const unit = document.getElementById('unit').value.trim() || '個';

// 修改後
const unit = '個';
```

#### 修復 2：刪除重複程式碼
- 刪除了 `{% block scripts %}` 中重複的 380+ 行程式碼
- 所有功能已經在 `{% block content %}` 中

#### 修復 3：加入等待機制
```javascript
// 等待 auth_ui.js 載入完成
let retries = 0;
while (!window.checkAuth && retries < 50) {
    await new Promise(resolve => setTimeout(resolve, 100));
    retries++;
}
```

#### 修復 4：加入除錯訊息
```javascript
console.log('[Product Create] Auth result:', authResult);
console.log('[Product Create] Setting createdBy to:', userName);
```
