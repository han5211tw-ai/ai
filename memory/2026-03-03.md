**問題**：卡片統計數字與實際列表不符

- 前端 `renderTable()` 使用 `needs.code` 前綴判斷狀態
- 後端列表查詢使用 `staging.status` 篩選
- 後端統計查詢也使用 `needs.code` 前綴

### 16. 庫存 Parser 欄位修正 ✅
**問題**：CP-AM-0018 庫存顯示為 0，但原始檔有數量

**原因**：
- CSV 欄位名稱為「現有庫存數量」
- Parser 尋找「庫存數量」，找不到則預設為 0

**修正**：`inventory_parser.py`
```python
# 修改前
qty_col = row.get('庫存數量', row.get(5, 0))

# 修改後
qty_col = row.get('現有庫存數量', row.get('庫存數量', row.get(5, 0)))
```

**結果**：
- CP-AM-0018（超微 RYZEN 7-7700 MPK 處理器）
  - 總公司倉庫：3 顆
  - 豐原門市：2 顆
  - 總庫存：**5 顆** ✅
