## 下午：銷貨資料修正與編輯功能實作

### 1. 銷貨資料錯誤修正

**問題：** DY-1150401-010 單號資料錯誤
- 產品名稱：「宏 AspireLite...」→ 應為「宏碁 AspireLite...」
- 產品編號：SE-NB（業務代號）→ 應為 NB-AC-0006
- 成本：0 → 應為 18,857
- 毛利：19,905 → 應為 1,048
- 毛利率：100% → 應為 5.27%

**修正 SQL：**
```sql
UPDATE sales_history 
SET product_name = '宏碁 AspireLite AL16-53P-53WA',
    product_code = 'NB-AC-0006',
    cost = 18857,
    profit = 1048,
    margin = 0.0527
WHERE sales_invoice_no = 'DY-1150401-010';
```

### 2. 銷貨輸入頁面編輯功能實作

**問題：** 原有 `enableEdit()` 函數只顯示「編輯功能開發中」，無實際功能

**實作內容：**

#### 後端 API (`app.py`)
新增 `/api/sales/update` 端點：
- 權限檢查：從 request body 讀取 `user_title`，僅限老闆/會計
- 先刪除原有明細，再重新插入
- 保留單據頭資訊（日期、客戶、業務員等）
- 檢查利潤：虧損品項禁止儲存

#### 前端功能 (`sales_input.html`)
新增功能：
- `renderEditModal()` — 渲染編輯模式介面
- `calculateEditRow()` — 即時計算單列金額/利潤
- `calculateEditTotal()` — 計算總計與虧損檢查
- `addEditRow()` — 新增產品項目
- `removeEditRow()` — 刪除產品項目
- `cancelEdit()` — 取消編輯，返回檢視模式
- `saveEdit()` — 儲存修改到資料庫

**編輯介面功能：**
- 可編輯：產品型號、產品名稱、數量、單價、成本
- 自動計算：金額、利潤、毛利率
- 安全檢查：虧損品項顯示警告並禁用儲存按鈕
- 必填驗證：產品名稱不能為空
