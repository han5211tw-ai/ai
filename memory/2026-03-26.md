## ERP AI 聊天機器人實作

### 功能概述
在 ERP 系統中新增 AI 聊天機器人，支援即時資料庫查詢和自然語言對話。

### 技術架構
- **模型**: oMLX Qwen3.5-9B-6bit（本機部署）
- **API 端點**: `http://127.0.0.1:8001/v1/chat/completions`
- **後端**: Python Flask + SSE 串流
- **前端**: 浮動視窗組件

### 實作內容

#### 1. 後端 API (`app.py`)
- 建立 `_OMLX_SESSION` Session（`trust_env = False` 避免 macOS 26 + Gunicorn fork crash）
- 路由 `POST /api/chat`，支援 SSE 串流回傳
- 兩階段對話流程：
  1. AI 生成 SQL 查詢
  2. 系統執行 SQL 並將結果回傳給 AI
  3. AI 生成最終自然語言回答
- 對話記錄寫入 SQLite `chat_logs` 資料表
- 安全限制：只允許 SELECT 查詢

#### 2. 前端組件 (`shared/ai-chat.js`)
- 右下角浮動機器人按鈕
- 點擊展開聊天視窗
- 支援多輪對話（完整 history 帶入）
- `sessionStorage` 產生 `sessionId`
- ReadableStream 偵測（支援串流 / fallback）
- 自動過濾思考區塊和 SQL（只顯示最終答案）
- Enter 換行，按鈕送出，送出後清空輸入框

#### 3. Gunicorn 設定 (`gunicorn.conf.py`)
- `worker_class = "gthread"`
- `threads = 4`
- `timeout = 180`（SSE 串流需要較長 timeout）

#### 4. System Prompt 設計
包含以下規則：
- 完整資料庫結構說明（sales_history, customers, inventory, staff_roster, purchase_history）
- SQLite 日期函數
- 常用查詢範例（業績、庫存、客戶、班表）
- 常見簡稱對照（總倉→總公司倉庫、門市→豐原/潭子/大雅等）
- 產品名稱智慧匹配（5060→RTX 5060 等）
- 輸出規則（只給答案，不顯示思考過程）

### 測試結果
- ✅ SSE 串流正常
- ✅ SQL 查詢自動執行
- ✅ 資料庫記錄正確寫入
- ✅ 思考過程和 SQL 已隱藏

## 推薦備貨頁面需求單修復

### 問題描述
推薦備貨商品頁面（`/recommended_products`）送出需求時出現「提交失敗」錯誤。

### 問題分析
API 使用了資料庫中不存在的欄位（`need_no`, `source`），導致 SQL INSERT 失敗。

### 修復內容
- 移除不存在的欄位
- 改用實際存在的欄位
- 新增重複檢查機制
- 新增 Telegram 通知

### 測試結果
- ✅ API 回應正常
- ✅ 需求單成功寫入資料庫
- ✅ Telegram 通知成功發送
