# 2026-03-26 工作記錄

## 推薦備貨頁面需求單修復

### 問題描述
推薦備貨商品頁面（`/recommended_products`）送出需求時出現「提交失敗」錯誤。

### 問題分析
發現有兩個版本的推薦備貨頁面：
1. `/recommended_products` - 使用 `dashboard-site/recommended_products.html`（有分類按鈕和數量選擇器）
2. `/電腦舖ERP系統/recommended_products.html` - 舊版卡片式介面

實際使用的是第一個版本，其呼叫 `/api/recommended-products/order` API。

### 根本原因
API 使用了資料庫中不存在的欄位：
- `need_no` 欄位不存在
- `source` 欄位不存在

導致 SQL INSERT 失敗。

### 修復內容

#### 1. 修正 `/api/recommended-products/order` API
**檔案**: `dashboard-site/app.py`

- 移除不存在的 `need_no` 和 `source` 欄位
- 改用實際存在的欄位：`date`, `product_code`, `item_name`, `quantity`, `department`, `requester`, `request_type`, `purpose`, `status`, `remark`, `created_at`
- 新增重複檢查：同一天、相同產品、相同申請人、狀態為「待處理」時跳過
- 改善錯誤訊息

#### 2. 新增 Telegram 通知
在 API 成功建立需求單後，發送 Telegram 通知給老闆：
- 通知標題：📦 **推薦備貨需求通知**
- 包含資訊：日期、填表人、部門、產品名稱、數量、需求筆數
- 發送對象：老闆個人 Telegram（與一般請購需求相同）
- 背景執行，不阻塞 API 回應

### 測試結果
- ✅ API 回應正常
- ✅ 需求單成功寫入資料庫
- ✅ 重複提交防護機制運作正常
- ✅ Telegram 通知成功發送

### 相關檔案
- `dashboard-site/app.py` - 後端 API
- `dashboard-site/recommended_products.html` - 前端頁面
