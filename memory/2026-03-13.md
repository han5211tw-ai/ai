## 技術筆記

### 登入流程時序問題
- `base.html` 和子頁面的 `DOMContentLoaded` 是並行的
- 子頁面不能依賴 `DOMContentLoaded` 時讀取 `localStorage`
- 應使用 `window.onLoginSuccess` 回調機制，確保登入完成後才執行頁面初始化

### 自動轉跳邏輯
```javascript
if (window.location.pathname !== '/' && window.location.pathname !== '/index.html') {
    window.location.href = '/';
}
```

### 5. 登入轉跳邏輯修正
- **問題**：登入後自動轉跳首頁的功能，導致從首頁點選單切換頁面時也會跳回首頁
- **原因**：轉跳邏輯放在 `requireLogin` 的 Promise resolve 中，每次檢查登入狀態都會執行
- **修復**：
  - `auth_ui.js`：新增 `_justLoggedIn` 標記區分「剛登入」和「已登入」
  - `base.html`：只在 `_justLoggedIn = true` 時才轉跳首頁
- **行為**：
  - 從書籤/分頁還原登入 → 轉跳首頁 ✅
  - 從首頁點選單切換頁面 → 不轉跳 ✅
