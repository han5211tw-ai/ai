#### 2. 修改 cron job 設定 (`~/.openclaw/cron/jobs.json`)
- 從 `agentTurn` 模式改為 `exec` 模式
- 直接執行 Python 腳本，不再經過 Agent 處理
- 將 timeout 從 120 秒延長到 180 秒（避免超時）

**修改前：**
```json
"payload": {
    "kind": "agentTurn",
    "message": "執行 ... 發送到 Telegram 工作群組",
    "timeoutSeconds": 120
}
```

**修改後：**
```json
"payload": {
    "kind": "exec",
    "command": "cd /Users/aiserver/.openclaw/workspace && python3 scripts/morning_report.py",
    "timeoutSeconds": 180
}
```

### 經驗教訓
1. **Agent 帳號 vs Bot 帳號**：Agent 發送的訊息可能帶有互動元素（如「更多詳情」按鈕），而 Bot 發送的是純粹的文字訊息
2. **cron job 模式選擇**：對於單純的執行任務，使用 `exec` 模式比 `agentTurn` 更直接、更可控
3. **發送機制**：需要純文字通知時，應直接使用 Bot API，而非透過 Agent 的 `message` 工具
