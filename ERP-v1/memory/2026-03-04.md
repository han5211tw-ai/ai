## 客戶資料管理原則（已定案）

**核心原則：「人員第一線篩選，系統輔助提示」**

| 情境 | 做法 |
|------|------|
| 新客戶來店 | 門市人員詢問姓名 + 手機，系統搜尋提示相似客戶 |
| 找到相似客戶 | 門市確認：「請問是手機 0912... 的黃先生嗎？」 |
| 客戶說換手機了 | 更新舊資料的手機欄位，不建新檔 |
| 確認是不同人 | 建立新客戶檔案（同名不同人） |
| 系統角色 | 提示可能重複，但不強迫合併 |

**不收集敏感資料：**
- ❌ 身分證字號
- ❌ 完整生日
- ❌ 詳細地址

**系統已實作功能：**
- ✅ 客戶搜尋支援姓名、手機、統編
- ✅ 相似名稱標記警示
- ✅ 建立新客戶前雙重確認機制
