## 📋 今日完成項目總覽

### 1️⃣ 需求表 Bug 修復（重大修復）

#### 問題 1：JavaScript 變數作用域錯誤（最嚴重）
- **現象**：送單按鈕卡在「提交中...」，無法完成提交
- **原因**：`items` 變數在 `try` 區塊內宣告，外部無法存取
- **修復**：將 `items` 移到 `try` 區塊外宣告

#### 問題 2：新品判斷邏輯缺陷
- **現象**：員工直接輸入品名時無法識別為新品
- **原因**：只依賴 `data-is-new` 屬性，但手動輸入時不會設置
- **修復**：增加自動檢測 `(!code && name)` 判斷邏輯

#### 問題 3：Staging API 超時
- **現象**：新品送單時 API 無回應導致卡住
- **修復**：加入 `AbortController` 5秒超時處理

#### 問題 4：驗證失敗按鈕鎖定
- **現象**：驗證失敗後按鈕永遠顯示「提交中...」
- **修復**：所有驗證失敗的 `return` 前重置 `isSubmitting`

#### 問題 5：重複提交報錯
- **現象**：重複資料導致 SQL 錯誤，前端卡住
- **修復**：改用 `INSERT OR IGNORE`，回傳清楚訊息

#### 問題 6：後端 API 錯誤處理
- **修復範圍**：`get_latest_needs`, `get_recent_needs`, `cancel_need`, `complete_need`, `update_need_remark`
- **統一模式**：`try-except-finally` 確保連線關閉
