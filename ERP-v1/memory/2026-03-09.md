## Telegram 通知崩潰問題解決方案 ✅

### 問題描述
macOS Python 多執行緒使用 `requests` 庫時，會觸發系統代理檢測（`_scproxy`），導致 Objective-C runtime 崩潰。

### 解決方案
在 `app.py` 開頭加入環境變數，禁止 Python 檢測 macOS 系統代理：

```python
import os
os.environ['NO_PROXY'] = '*'
os.environ['no_proxy'] = '*'
```

### 原理
- `requests` 庫看到 `NO_PROXY=*` 會跳過所有代理檢測
- 不會呼叫 `_scproxy` 模組
- 不會觸發 Objective-C runtime
- 背景執行緒可以正常運作

- **20:58** 確認後續需求通知都正常收到
- 無崩潰報告
- 通知發送與需求提交同步完成
