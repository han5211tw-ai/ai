### 1. 需求表「取消後重打」Bug 修復 ✅

**問題描述：**
員工打需求單送出後，因客戶名稱沒有打到所以取消重打，再送出時會出現「資料已存在」錯誤。

**根本原因：**
- `needs` 資料表有 `UNIQUE(date, item_name, quantity, requester, product_code)` 限制
- 取消需求是「軟刪除」，資料仍留在資料庫，只是標記 `status='已取消'` 和 `cancelled_at` 時間
- 重打相同資料時，觸發 UNIQUE 限制，被 `INSERT OR IGNORE` 忽略

**修復方式：**
修改 `/api/needs/batch` API 邏輯（`app.py` 約第 2980 行）：
1. 插入前先檢查是否為「已取消」的資料（`cancelled_at IS NOT NULL`）
2. 如果是已取消的資料，則「復活」該筆記錄：
   - 更新 `status = '待處理'`
   - 清空 `cancelled_at = NULL`
   - 更新 `created_at` 和其他相關欄位
3. 如果不是已取消的資料，則維持原來的 `INSERT OR IGNORE` 邏輯

**修改檔案：**
- `/Users/aiserver/.openclaw/workspace/dashboard-site/app.py`

**重啟服務：**
```bash
cd /Users/aiserver/.openclaw/workspace/dashboard-site
./manage_gunicorn.sh restart
```
